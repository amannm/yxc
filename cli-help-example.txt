yamaha-xca 1.0.0
CLI for Yamaha Extended Control API (XCA) / MusicCast receivers

USAGE:
  yxc [GLOBAL OPTIONS] <command> [COMMAND OPTIONS] [ARGS...]

DESCRIPTION:
  Interact with Yamaha receivers that expose the Extended Control API over HTTP.
  By default, requests are GET unless the endpoint requires POST.

  Most endpoints return JSON with:
    response_code: 0 indicates success, non-zero indicates an error/status.

GLOBAL OPTIONS:
  -H, --host <host>              Receiver host/IP (e.g. 192.168.1.50) [required unless --base-url]
      --base-url <url>           Full base URL (default: http://<host>/YamahaExtendedControl)
      --api-prefix <path>        API prefix (default: /v1)
      --zone <zone>              Default zone for zone commands: main|zone2|zone3|zone4 (default: main)

      --timeout <duration>       Request timeout (e.g. 2s, 500ms) (default: 5s)
      --retries <n>              Retry count on transient network errors (default: 0)

      --auth <user:pass>         Basic auth (rare; device dependent)
      --header <k:v>             Add an HTTP header (repeatable)

      --dry-run                  Print the HTTP request that would be sent, do not send it
      --curl                     Output an equivalent curl command
      --format <fmt>             Output: json|pretty|yaml|table (default: pretty)
      --jq <filter>              Apply jq filter to JSON output (requires jq installed)

  -v, --verbose                  Verbose logging (repeatable: -vv for more)
  -q, --quiet                    Only print command output (no status lines)
      --no-color                 Disable ANSI colors
      --version                  Print version
  -h, --help                     Print help
      --help-all                 Print full help including all commands

EXIT CODES:
  0   Success (response_code == 0, or command succeeded)
  2   CLI usage error / invalid args
  3   HTTP/network error (timeout, connection refused, etc.)
  4   API returned non-zero response_code
  5   JSON parse/format error

EXAMPLES:
  # Zone status
  yxc -H 192.168.1.50 zone status --zone main

  # Power on Zone 2
  yxc -H 192.168.1.50 zone power on --zone zone2

  # Set volume to 45
  yxc -H 192.168.1.50 zone volume 45

  # Volume up by 3 steps
  yxc -H 192.168.1.50 zone volume up --step 3

  # Set input and disable autoplay behavior
  yxc -H 192.168.1.50 zone input hdmi1 --mode autoplay_disabled

  # Net/USB: list 8 items from index 0 for input "net_radio"
  yxc -H 192.168.1.50 netusb list --input net_radio --index 0 --size 8

  # Net/USB: search
  echo '{"string":"Miles Davis"}' | yamaha-xca -H 192.168.1.50 netusb search --list-id main --stdin

  # System reboot (careful)
  yxc -H 192.168.1.50 system reboot --scope system

COMMANDS:
  system        System-level controls (dimmer, HDMI outs, party mode, reboot, etc.)
  zone          Zone controls (power, volume, input, sound, cursor/menu, etc.)
  tuner         Tuner controls (band, frequency, presets, scans)
  netusb        Network/USB playback & browsing (playback, list control, presets, recent)
  cd            CD playback controls
  clock         Clock/alarm settings
  dist          MusicCast Link distribution/group controls
  raw           Call an arbitrary endpoint by path (escape hatch)

Run 'yamaha-xca <command> --help' for command-specific help.

RAW COMMAND
USAGE:
  yxc raw <METHOD> <PATH> [--query <k=v>]... [--data <json>] [--stdin]

EXAMPLES:
  yxc -H 192.168.1.50 raw GET /v1/main/getStatus
  yxc -H 192.168.1.50 raw GET /v1/system/getLocationInfo
  yxc -H 192.168.1.50 raw POST /v1/netusb/setSearchString --data '{"list_id":"main","string":"Radiohead"}'

NOTES:
  PATH may be absolute (/v1/...) or relative (system/getLocationInfo).
  When --stdin is used, request body is read from STDIN.

RESPONSE CODES (response_code):
  0       Successful request
  1       Initializing
  2       Internal Error
  3       Invalid Request
  4       Invalid Parameter
  5       Guarded
  6       Time Out
  99      Firmware Updating
  100-115 Streaming service errors
  200-201 Distribution errors
